确保第一行包含以下列名（必须完全一致）：

| A列 | B列 | C列 | D列 | E列 | F列 |

## ❌ 如果失败了

### 查看错误信息

在 Actions 日志中找到红色的 ❌ 标记，查看错误信息。

### 常见错误和解决方法

#### 错误 1：Twitter 登录失败

```
❌ Twitter登录失败: 401 Unauthorized
```

**原因**：Twitter API 密钥错误或权限不足

**解决方法**：
1. 检查 GitHub Secrets 中的 Twitter 密钥是否正确
2. 确认 Twitter 应用的权限是 "Read and Write"
3. 重新生成 Access Token 并更新 Secrets

---

#### 错误 2：Google Sheet 连接失败

```
❌ 连接Google Sheets失败
```

**原因**：服务账号没有访问权限

**解决方法**：
1. 打开 Google Sheet
2. 点击 "共享"
3. 确认服务账号邮箱已添加
4. 确认权限是 "编辑者"

---

#### 错误 3：没有找到未发送文章

```
❌ 没有找到未发送的文章
```

**原因**：所有文章都已发送，或者没有标记为"未发送"的文章

**解决方法**：
1. 在 Google Sheet 中添加新文章
2. 确保发送状态填"未发送"（中文）

---

#### 错误 4：推文超过字符限制

```
⚠️ 推文超过280字符，将自动截断
```

**原因**：标题太长

**解决方法**：
- 系统会自动截断，这是正常的
- 建议标题控制在 150 字符以内

---

#### 错误 5：找不到文件

```
ModuleNotFoundError: No module named 'xxx'
```

**原因**：代码文件缺失或路径错误

**解决方法**：
1. 检查 GitHub 仓库的文件结构
2. 确保以下文件存在：
   - `send_post.py`
   - `.github/workflows/twitter-bot.yml`
   - `src/tools/google_sheets_tool.py`
   - `src/tools/twitter_tool.py`
   - `src/utils/post_utils.py`
